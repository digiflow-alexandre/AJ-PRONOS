Flow utilisateur

  1. Écran 1 — État vide : tap sur "+ Ajouter une condition"
  2. Écran 2 — Choisir un type parmi les ~18 disponibles (grille 2 colonnes)
  3. Écrans 3-8 — Selon le type choisi, des sous-options apparaissent (segmented controls + chips + inputs selon le besoin)
  4. Écran 5/9 — La condition apparaît dans la liste, on peut en rajouter d'autres (combiné AND), supprimer une condition, ou éditer
  5. Preview dynamique — Le texte canonique se construit en bas. C'est exactement ce qui sera passé au tracker auto.
  6. Bouton "Suivant" en haut à droite → écran cote / confiance / raisonnement / publication (inchangé)

Points-clés design

  • Match toujours visible en haut (carte or) — l'admin sait pour quel match il pronostique
  • Couleurs respectent la DA : noir/or/crème, accent or sur les choix actifs
  • Conditions empilées verticalement avec un connecteur "+ ET +" entre chacune — visuellement explicite
  • Preview verte si le tracker reconnaît tout, jaune si une condition n'est pas trackable auto
  • Tous les patterns supportés par notre track-results sont accessibles via picker structuré
